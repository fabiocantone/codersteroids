# Code Standards

## Defaults

- Every file should have one clear responsibility.
- Treat 250-400 lines as an attention threshold, not an automatic failure.
- Split a file when it mixes responsibilities, not merely because it is long.
- Public APIs need types, tests, and documentation.
- New dependencies need a recorded reason.
- Prefer existing project patterns over generic defaults.

## Folder Documentation

Significant folders should include `README.md` or `FOLDER.md` describing:

- Purpose.
- Main files.
- Local patterns.
- What does not belong there.
- Relevant flows.
